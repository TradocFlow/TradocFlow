Kanban Board Test Page

Testing Instructions:

1. Start the API server: cargo run --bin api_server

2. Click the link below to open the Kanban board

3. The board will load with a default project ID for testing

Open Kanban Board (Test Project)

Features Implemented:

  • Responsive Kanban board layout with drag-and-drop
  • Task creation, editing, and status management
  • Real-time updates using Server-Sent Events
  • Search functionality for tasks
  • Filtering by assignee and priority
  • Visual priority indicators and due date tracking
  • Modal-based task editing interface
  • Toast notifications for user feedback
  • Keyboard shortcuts (Ctrl+N for new task, Esc to close modal)
  • Mobile-responsive design

API Endpoints:

  • GET /api/projects/{id}/kanban - Get board data
  • POST /api/projects/{id}/kanban - Create new task
  • GET /api/kanban/{id} - Get specific task
  • PUT /api/kanban/{id} - Update task
  • DELETE /api/kanban/{id} - Delete task
  • PUT /api/kanban/{id}/move - Move task between columns
  • GET /api/projects/{id}/events - Server-Sent Events stream

Note:

This is a test interface. In the actual application, the project ID would be passed from the main Slint application when opening the web-based Kanban board.
